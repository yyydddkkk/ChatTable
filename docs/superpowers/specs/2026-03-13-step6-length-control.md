# Step 6: 长度控制层设计

## 目标
实现五级长度控制 + 自然语言触发

## 五级长度定义

| 等级 | 描述 | Prompt 注入 |
|------|------|-------------|
| 1 | 极短 | "请用一句话回答，最多 20 字" |
| 2 | 较短 | "请简洁回答，50 字以内" |
| 3 | 中等 | "正常长度回答" |
| 4 | 较长 | "请详细回答，充分说明" |
| 5 | 极长 | "请尽可能详细地回答，包含所有细节" |

## 自然语言触发

- **变长**: "详细点" / "多说点" → level + 1
- **变短**: "简短点" / "少说点" → level - 1

## WebSocket 消息

- 客户端发送: `{ "type": "set_length", "level": 3 }`
- 服务端保存到会话上下文

## 文件变更

| 文件 | 职责 |
|------|------|
| `backend/app/core/length_control.py` | 长度控制核心逻辑 |
| `backend/app/main.py` | 集成长度控制 |
| `frontend/src/components/LengthAdjuster.tsx` | 前端调节器 |

## 验收标准
- [ ] 可调节长度等级（1-5）
- [ ] Agent 回复长度符合预期
- [ ] 说"详细点"后回复变长
- [ ] 前端构建成功
